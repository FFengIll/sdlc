# SDLC Skills System v3.2 - Flow-Based Design

**Date**: 2026-03-08
**Version**: 3.2 (Flow-Based)
**Status**: Design

## Overview

基于流程的 SDLC 技能系统，强调阶段之间的关联和顺序，支持完整的开发工作流。

## 阶段定义

| 阶段 | 目的 | 检查内容 |
|-----|------|---------|
| **test** | 通过测试来验证 | Lint, 类型检查, 单元测试, 集成测试, E2E |
| **verify** | 验证符合 spec | 需求覆盖, API 匹配, 功能完整 |
| **secure** | 安全检查 | 漏洞扫描, 依赖检查, 密钥检测 |
| **cr** | 代码审查 | 最佳实践, 架构, 可维护性 |

```
test    → 代码能跑通吗？ (机器检查)
verify  → 功能做对了吗？ (对比 spec)
secure  → 有安全问题吗？ (安全扫描)
cr      → 代码质量好吗？ (人工审查)
```

## 技能详细定义

### 质量保证技能

#### `/sdlc test`
**目的**: 通过测试来验证代码能跑通

**检查内容**:
```bash
/sdlc test [check]

checks:
  lint       # 代码风格检查
  typecheck  # 类型检查
  format     # 代码格式化
  unit       # 单元测试
  integ      # 集成测试
  e2e        # E2E 测试
  coverage   # 覆盖率分析
  all        # 全部检查 (默认)
```

**执行顺序**:
1. lint (快速失败)
2. format (自动修复)
3. typecheck (类型验证)
4. unit (单元测试)
5. integ (集成测试)
6. e2e (端到端测试)
7. coverage (覆盖率报告)

**输出**: 测试报告

**示例**:


## 工作流流程

### 功能开发
```
/sdlc start feature
    ↓
research → spec → coding → test → verify → secure → cr → commit → pr
    ↓
  MERGE
```

### Bug 修复
```
/sdlc start bugfix
    ↓
debug → coding → test → verify → secure → commit → pr
    ↓
  MERGE
```

### 重构
```
/sdlc start refactor
    ↓
cr → spec → coding → test → verify → secure → cr → commit → pr
    ↓
  MERGE
```

## 使用示例

### 完整流程
```bash
# 1. 开始
/sdlc start feature "用户认证"
# → research → spec → coding

# 2. 编码完成后
/sdlc next
# → test (lint + typecheck + format + unit + integ + e2e)
/sdlc next
# → verify (验证符合 spec)
/sdlc next
# → secure (安全检查)
/sdlc next
# → cr (代码审查)

# 3. 提交
/sdlc next
# → commit → pr → MERGE
```

### 快速测试
```bash
# 运行所有测试
/sdlc test

# 只运行单元测试
/sdlc test unit

# 只检查 lint 和类型
/sdlc test lint typecheck
```

### 验证需求
```bash
# 验证实现符合 spec
/sdlc verify

# 验证指定 spec
/sdlc verify ./docs/spec/user-auth.md
```

## 关键改进

1. **简化流程**: test 合并了 validate，减少一个阶段
2. **清晰分离**:
   - test: 代码能跑通吗？ (机器检查)
   - verify: 功能做对了吗？ (对比 spec)
3. **test 包含**:
   - 代码质量检查 (lint, typecheck, format)
   - 功能测试 (unit, integ, e2e)
   - 覆盖率分析 (coverage)

## 与 v3.1 的变化

| v3.1 | v3.2 | 说明 |
|-----|------|------|
| coding → **validate** → **verify** → test → ... | coding → **test** → **verify** → ... | test 合并了 validate |
| /sdlc validate | /sdlc test | 合并到 test |
| /sdlc test | /sdlc test | 保留，扩展功能 |
