# SDLC Skills System v3 - Flow-Based Design

**Date**: 2026-03-08
**Version**: 3.0 (Flow-Based)
**Status**: Design

## Overview

基于流程的 SDLC 技能系统，强调阶段之间的关联和顺序，支持完整的开发工作流。

## 核心概念

## 技能定义

### 主入口

#### `/sdlc`
**用法**:
```
/sdlc <command> [args]

Commands:
  start <type>     - 开始新流程 (feature|bugfix|refactor|research)
  status           - 查看当前状态
  next             - 执行下一阶段
  skip <phase>     - 跳过阶段
  phase <name>     - 跳转到指定阶段
  end              - 结束当前流程
```

**状态管理**:
```
.sdlc/
└── state.json
{
  "workflow": "feature",
  "current_phase": "spec",
  "completed": ["research"],
  "branch": "feature/user-auth",
  "started_at": "2026-03-08T10:00:00Z"
}
```

---

### 流程控制技能

#### `/sdlc start`
**用途**: 开始一个新的工作流

**用法**:
```
/sdlc start <type> [description]

type:
  feature    - 功能开发
  bugfix     - Bug 修复
  refactor   - 重构
  research   - 调研
```

**功能**:
- 创建工作分支
- 初始化状态文件
- 引导进入第一个阶段
- 可选：创建 issue/ticket

---

#### `/sdlc status`
**用途**: 查看当前流程状态

**输出**:
```
当前工作流: feature
当前阶段: spec
已完成阶段: research
分支: feature/user-auth
开始时间: 2026-03-08 10:00

下一阶段: coding

建议:
- 确认 spec 文档完整性
- 检查设计图是否完成
- 准备开始编码
```

---

#### `/sdlc next`
**用途**: 执行流程的下一阶段

**功能**:
- 自动识别当前阶段
- 调用相应的阶段技能
- 更新状态
- 完成后提示下一步

---

#### `/sdlc skip`
**用途**: 跳过当前阶段

**用法**:
```
/sdlc skip [phase]
```

**示例**:
```
# 跳过当前阶段
/sdlc skip

# 跳过指定阶段
/sdlc skip research
```

---

### 阶段技能

#### `/sdlc research`
**阶段**: 调研
**下一步**: spec

**功能**:
- 技术调研
- 方案比较
- 生成调研文档

**输出**: `./docs/research/{date}-{topic}.md`

**完成条件**:
- [ ] 调研文档已创建
- [ ] 方案对比完成
- [ ] 技术选型确定

---

#### `/sdlc spec`
**阶段**: 规格
**上一步**: research
**下一步**: coding

**功能**:
- 需求分析
- 架构设计
- API 设计
- 可调用 `/pencil` 画图

**输出**: `./docs/spec/{date}-{title}.md`

**完成条件**:
- [ ] Spec 文档已创建
- [ ] 需求已明确
- [ ] 设计已完成
- [ ] API 已定义

---

#### `/sdlc coding`
**阶段**: 编码
**上一步**: spec
**下一步**: validate

**功能**:
- 编码指导
- 代码生成辅助
- 进度跟踪

**注意**: 这是人工编码阶段，技能提供辅助

**完成条件**:
- [ ] 功能已实现
- [ ] 代码自审完成
- [ ] 注释完整

---

#### `/sdlc validate`
**阶段**: 验证
**上一步**: coding
**下一步**: secure

**功能**:
- Lint 检查
- 类型检查
- 代码格式化

**完成条件**:
- [ ] Lint 通过
- [ ] 类型检查通过
- [ ] 代码已格式化

---

#### `/sdlc secure`
**阶段**: 安全
**上一步**: validate
**下一步**: test

**功能**:
- 安全漏洞扫描
- 依赖检查
- 密钥检测

**完成条件**:
- [ ] 无安全漏洞
- [ ] 依赖安全
- [ ] 无密钥泄露

---

#### `/sdlc test`
**阶段**: 测试
**上一步**: secure
**下一步**: cr

**功能**:
- 运行测试
- 生成测试
- 覆盖率分析

**子命令**:
```
/sdlc test unit      - 单元测试
/sdlc test integ     - 集成测试
/sdlc test e2e       - E2E 测试
/sdlc test coverage  - 覆盖率
```

**完成条件**:
- [ ] 单元测试通过
- [ ] 覆盖率达标
- [ ] 无失败测试

---

#### `/sdlc cr` (Code Review)
**阶段**: 审查
**上一步**: test
**下一步**: commit

**功能**:
- 代码审查
- 最佳实践检查
- 改进建议

**输出**: 审查报告

**完成条件**:
- [ ] 审查完成
- [ ] 问题已修复
- [ ] LGTM 获得

---

#### `/sdlc commit`
**阶段**: 提交
**上一步**: cr
**下一步**: pr

**功能**:
- 生成 commit message
- 提交代码
- 打标签（可选）

**完成条件**:
- [ ] Commit message 清晰
- [ ] 代码已提交
- [ ] 分支已推送

---

#### `/sdlc pr`
**阶段**: PR
**上一步**: commit
**下一步**: MERGE

**功能**:
- 创建 PR
- 生成 PR 描述
- 关联 issue

**完成条件**:
- [ ] PR 已创建
- [ ] 描述完整
- [ ] Reviewer 已指定

## 使用示例

### 开始功能开发
```bash
# 1. 开始
/sdlc start feature "用户认证系统"
# → 创建分支 feature/user-auth
# → 状态: research

# 2. 执行调研
/sdlc next
# → 执行 /sdlc research
# → 生成调研文档

# 3. 编写规格
/sdlc next
# → 执行 /sdlc spec
# → 可调用 /pencil 画图

# 4. 编码
/sdlc next
# → 提示开始编码
# → [人工编码]

# 5. 继续（编码完成后）
/sdlc next
# → 执行 /sdlc validate
/sdlc next
# → 执行 /sdlc secure
/sdlc next
# → 执行 /sdlc test
/sdlc next
# → 执行 /sdlc cr

# 6. 提交和 PR
/sdlc next
# → 执行 /sdlc commit
/sdlc next
# → 执行 /sdlc pr

# 7. 完成
/sdlc end
# → 清理状态
```

### 快速跳转
```bash
# 查看状态
/sdlc status

# 跳到特定阶段
/sdlc phase test

# 跳过当前阶段
/sdlc skip
```

## 与现有技能的映射

| 现有技能 | 新位置 | 说明 |
|---------|--------|------|
| `/spec` | `/sdlc spec` | 作为阶段的一部分 |
| `/pencil` | `/pencil` | 基础技能，spec 可调用 |
| `/pr` | `/sdlc pr` | 作为阶段的一部分 |
| `/codereview` | `/sdlc cr` | 作为阶段的一部分 |
| `/git-commit` | `/sdlc commit` | 作为阶段的一部分 |
| `/research` | `/sdlc research` | 作为阶段的一部分 |
| `/test-go` | `/sdlc test` | 合并 |
| `/refactor-go` | `/sdlc cr` | 合并 |
| `/codeclean` | `/sdlc validate` | 合并 |

## 关键改进

1. **流程导向**: 不再是孤立技能，而是完整的流程
2. **状态管理**: 跟踪当前阶段和进度
3. **自动流转**: `/sdlc next` 自动执行下一步
4. **灵活跳转**: 可跳过或跳转到任意阶段
5. **完成条件**: 每个阶段有明确的完成标准
6. **工作流模板**: 预定义常用流程
